# Research Agent Workflow

本文档定义 V8 研究代理的标准处理流程，用于处理一条产业新闻、公司事件或财报信息，并输出研究结论。

## 1. 输入类型

- 产业新闻
- 公司公告
- 财报摘要
- 产品发布
- 供应链消息

## 2. 标准处理流程

1. 识别主题
2. 匹配观察池
3. 映射产业链节点
4. 判断是否构成催化剂
5. 更新 `order_confirmation_score`
6. 更新 `strategic_score`
7. 输出研究结论

## 3. 处理原则

- 先识别事件属于哪个主题或多个主题
- 再判断该事件是否触发核心观察池中的公司
- 结合产业链位置判断事件是核心催化剂还是间接影响
- 如果事件推动订单、收入、客户验证或产线验证，则提升 `order_confirmation_score`
- `strategic_score` 只用于研究排序，不用于直接买卖判断

## 4. 输出格式

研究代理在输出时应包含以下字段：

- 事件摘要
- 涉及主题
- 相关公司
- 产业链影响
- 催化剂强度
- 订单验证程度
- 风险点
- 研究结论
- 是否进入重点观察

## 5. 结果判定

### 5.1 催化剂

当事件满足以下任一条件，可视为候选催化剂：

- 明确的订单落地
- 收入开始兑现
- 客户验证完成
- 产品发布进入商业化阶段
- 供应链约束被缓解或强化

### 5.2 重点观察

以下情况应进入重点观察：

- 事件直接触发核心主题
- 事件影响核心观察池 A 级公司
- 订单验证信号明显增强
- 产业链节点出现实质变化

## 6. 研究约束

- 不输出买卖建议
- 不预测股价
- 不接入外部 API
- 不将短期情绪等同于基本面验证

## 7. 与评分系统的关系

- `order_confirmation_score` 反映事件是否从故事进入验证阶段
- `strategic_score` 反映 12-24 个月产业趋势强度
- 研究代理输出的结论应回写到研究报告和观察池更新流程中
